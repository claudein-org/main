# This is NOT the Next.js you know

This is Next.js 16.2.9. It has breaking changes ‚Äî APIs, conventions, and file structure may all differ from your training data. Read the relevant guide in `node_modules/next/dist/docs/` before writing any code. Heed deprecation notices.

## Commands

Use Bun: `bun run dev` (dev), `bun run build`, `bun run start`. `bun run dev-https` runs the dev server over HTTPS (needed for OAuth callback testing).

## Environment

`CLIENT_ID` and `CLIENT_SECRET` (LinkedIn OAuth) are validated at startup by `lib/env.ts` via zod. Import `env` from there rather than reading `process.env` directly.

## Styling

Styling uses **Panda CSS** (build-time, zero-runtime). You may author styles in **only these three files** ‚Äî never inline `style={{...}}`, never `.module.css`, never any other `.css`/`.css.ts`:

- `css/global.css.ts` ‚Äî semantic, element-level global styles via `defineGlobalStyles`. Applied to **elements** (`body`, `main`, `footer`, `a`, `h1`, `code`, `input`, `textarea`, `select`, `label`, `button`, ‚Ä¶), never to classes. Imported by `panda.config.ts` as `globalCss`.
- `css/layout.css.ts` ‚Äî **layout** utility classes via `css()`: `row`/`col` (flex direction), `align.{start,center,end}` (alignItems), `justify.{start,center,end,between}` (justifyContent), `gap.{xs,sm,md,lg,xl}`, `textAlign.center`, `w.full`, `grow`.
- `css/style.css.ts` ‚Äî **style** utility classes via `css()` / `cva()`: `font.size.{sm,base,lg}`, `font.weight.{normal,medium,bold}`, `tracking.{sm,tight}`, `color.{text,muted,claude,linkedin}`, `muted` (shorthand), `logo`, `btn` cva (variants: `color={dark,linkedin,claude}`).

Rules:
- Design tokens (brand colors, fonts) live in `panda.config.ts` `theme.extend.tokens` ‚Äî the single source of truth. Reference them by name (e.g. `color: "claude"`, `fontFamily: "sans"`).
- Compose utilities on elements with `cx(...)` from `@/styled-system/css`.
- `styled-system/` is generated by `panda codegen` (gitignored). `dev`, `dev-https`, and `build` run `panda codegen` first; never edit `styled-system/` by hand.
- `css/panda.css` is the Panda layer entry (`@layer reset, base, tokens, recipes, utilities;`), imported once in `app/layout.tsx`. It contains no authored styles.
- Panda runs via its PostCSS plugin (`postcss.config.cjs`), so the build stays on **Bun + Turbopack** ‚Äî no toolchain change. (Vanilla-extract was evaluated and rejected: its `.css.ts` compiler only works under Node + webpack, not Bun/Turbopack.)
